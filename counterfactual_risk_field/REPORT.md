# 从全局边界到局部风险场：反事实邻域几何用于多模态越狱检测

## 材料说明

- 评估对象：`intentguard_refactor/RCS_COUNTERFACTUAL_NEIGHBORHOOD_RISK_FIELD.md`
- 评估边界：本报告只判断 proposal 内部的创新性、方法学完整性和工程可行性，不据此宣称已完成穷尽式相关工作检索。
- 方法版本：Counterfactual Neighborhood Risk Field，简称 **CNRF v1**。
- 验证状态：算法与实验协议已经实现并通过合成测试；基于目标 benchmark 的 LLM 反事实数据尚待生成、人工审核和正式模型前向提取。

## 摘要

本工作研究一个现实但尚未解决的问题：多模态大模型的有害输入与无害输入，可能并不存在一个能跨数据集、模态和攻击载体稳定迁移的全局表征边界。已有表征检测器在训练来源上往往表现很好，但如果每个数据集只对应一个标签，检测器可能读取的是数据格式、模板、图片风格或来源水印，而不是风险本身。

CNRF 不训练新的神经投影。它以受控的有害—无害反事实对为基本单位：同一语义种子、同一载体和近似相同的外层形式构成一支从安全端点指向危险端点的局部风险箭头。测试输入先检索相似反事实中点，再读取自己沿附近风险箭头所处的位置。文本与图文分别建库，多个隐藏状态读出使用相同的无参数规则，最终门槛只由独立无害校准集确定。

仅从 proposal 判断，这项工作具有成为强论文的潜力，但当前创新性属于**有条件成立**：端点距离差、反事实配对和 K 近邻本身都不是足够的新贡献。论文必须同时证明三个命题：配对箭头确实比原始表示更少携带来源信息；局部风险场在等数据条件下优于 RCS 和普通双侧近邻；同一冻结方法同时满足低误拒与强 OOD 攻击高召回。任何一个命题失败，都应收缩论文主张。

## 一、审稿式结论

### 1.1 创新性评价

**结论：中高潜力，但尚不能仅凭公式认定创新性。**

proposal 的真正新意不在于提出一个新的距离函数，而在于改变风险几何的建模单位：

1. 从“全局两类边界”改为“内容—载体流形上的局部状态变化”；
2. 用反事实中点定位上下文，用成对箭头判别风险，使来源信息只参与检索而不直接决定标签；
3. 将来源水印消除从口头动机变成可证伪的机制实验，即比较原始端点、中点和箭头的来源可读性；
4. 把低误报工作点、最差来源性能和邻域覆盖不足作为主结果，而非只报告总体 AUROC。

如果上述证据链成立，贡献可被定位为“新的风险表征假设 + 对应无参数方法 + 反事实数据协议 + OOD 低误报证据”。如果实验只显示 CNRF 比 RCS 高一两个百分点，却没有来源可读性下降和等数据对照，贡献会退化成局部 KNN 的工程变体。

### 1.2 可行性评价

**结论：工程上可行，数据构造是主要瓶颈，算力不是。**

风险场拟合只需归一化、近邻检索和分位数校准，复杂度明显低于训练百万参数投影。仓库已有 Qwen2.5-VL-7B、Gemma3-12B、多层隐藏状态、多读出、RCS 复现以及目标攻击入口，因此模型侧基础设施充分。

最大风险来自反事实数据，而不是算法：

- 随意把 Alpaca 问题改成有害问题，通常会改变主题而非只改变风险变量；
- 只从 AdvBench 生成安全改写，虽然能形成干净箭头，却可能缺乏真实无害内容区域覆盖；
- 视觉语义型风险不能总在保持同一张图的同时完成安全切换；
- LLM 生成的“安全反事实”可能只是模板化拒绝，导致新的生成器水印；
- 同一语义种子的多个载体版本若跨 split 或在近邻中重复计数，会严重高估结果。

因此，本方案把 Alpaca/VizWiz 改为载体和真实无害校准来源，把 AdvBench/MM-SafetyBench 作为风险语义种子；LLM 只提出请求对，不生成回答；所有候选必须人工审核后才能进入正式参考库。

### 1.3 最强反对意见

一个严格审稿人最可能提出：

> 性能提升可能完全来自更干净、更平衡的成对数据，而不是局部风险场。若把同一批反事实数据交给 RCS、线性探针或普通双侧 KNN，它们也可能获得相同提升；所谓“来源抵消”还可能只是 LLM 生成模板趋同。

这个反对意见不能靠文字回应，只能靠两项实验解决：

1. **等数据机制比较**：所有方法使用相同语义包、相同端点、相同 split 和相同无害校准集；
2. **来源可读性审计**：在按 `pack_id` 分组的交叉验证下，用同一轻量分类器分别预测原始端点、中点和箭头的语义来源与载体来源。

若箭头来源可读性没有显著下降，或者 RCS 在等数据条件下不弱于 CNRF，应否定或显著收缩核心机制主张。

## 二、研究问题与可证伪假设

主研究问题是：

> 多模态大模型内部是否缺少稳定的全局风险边界，却存在可由少量结构化反事实配对恢复的局部风险变化场？

四个预注册假设如下：

- **H1，来源抵消**：在相同分组交叉验证下，风险箭头预测语义来源和载体来源的 macro-F1 显著低于原始端点与中点。
- **H2，局部性**：在完整来源或完整载体留出下，CNRF 在无害 FPR 不超过 5% 时的最差组 TPR 高于全局均值箭头、普通双侧 KNN 和 RCS。
- **H3，样本效率**：在相同独立 `pack_id` 数量下，CNRF 的优势在 5/10/20/40/80/160 个参考包曲线上已经出现，而不是依赖大量模板复制。
- **H4，多读出互补**：最后位置、非图像位置均值和图像位置均值的融合，能够提高 FigStep、JOOD、CS-DJ 的最差攻击召回，而不只是提高同分布验证集。

## 三、数据角色与冻结划分

### 3.1 数据源角色

| 角色 | 数据源 | 用途 |
|---|---|---|
| 风险语义种子 | AdvBench、MM-SafetyBench | 提供已经审核的有害请求，不让生成模型自由创造新的有害目标 |
| 文本载体与真实无害 | Alpaca | 提供语气、长度和外层请求形式；未使用样本进入独立无害校准 |
| 图文载体与真实无害 | VizWiz-VQA | 提供真实图像背景与视觉问答载体；未使用样本进入独立无害校准 |
| 冻结测试无害 | XSTest-safe、MM-Vet | 评价敏感但安全问题与真实多模态 utility 的误报/误拒 |
| 冻结测试有害 | XSTest-unsafe、JailBreakV-28K | 评价文本有害意图和多种越狱载体 |
| 强 OOD 攻击 | FigStep、JOOD、CS-DJ | 只在方法冻结后评估，不参与生成、选层、近邻数或门槛选择 |

### 3.2 包、对和端点

- `pack_id`：一个基础风险语义种子。它是 split、bootstrap 和样本效率统计的最小独立单位。
- `pair_id`：同一 `pack_id` 在一个具体载体上的一对端点。
- endpoint：有害请求或安全反事实请求。

同一 `pack_id` 的文本载体、VizWiz 图像载体、原生 MM-SafetyBench 图像载体及其所有派生版本必须处于同一个 split。检索时同一 `pack_id` 最多贡献一支箭头，避免一个语义种子的多种渲染淹没其他语义区域。

### 3.3 内部开发划分

由训练来源生成的包按 `pack_id` 一次性划分为：

- reference 70%：建立风险场；
- validation 15%：选择每个读出的层，固定 `k=5`；
- calibration 15%：独立确定最终门槛和邻域支持半径。

外部测试数据不进入上述任何阶段。原始图像及其裁剪、拼接、加字版本，外层模板和载体家族都必须跟随 `pack_id` 隔离。

## 四、反事实数据生成协议

### 4.1 生成任务

LLM 接收一个已知有害请求、其底层语义和一个载体风格提示，只输出 JSON 格式的请求对：

- `harmful_request`：复制或最小化改写原始有害请求，只能是请求，不能包含回答；
- `benign_counterfactual`：保持主题、实体、语言、语气、长度和载体，把请求行为改为预防、检测、报告、恢复或高层教育；
- 风险域、唯一改变的风险变量和最小性理由；
- `contains_answer=false`。

生成模型禁止扩写操作步骤、代码、数量或任何比种子更可执行的内容。它不负责最终标签，只负责提出候选。

### 4.2 载体构造

每个语义种子至少形成：

1. 一个 Alpaca 风格文本对；
2. 一个共享 VizWiz 图像背景的图文对；
3. 若种子具有原生图像，再形成一个共享原生图像的图文对。

第一版不声称所有视觉风险都能在完全相同像素下切换。对于风险主要由图片语义承载的样本，必须单独标记为 visual-semantic pair，并在人工审核中确认布局、图像风格和非风险变量是否足够匹配；不能把这类对与严格 carrier-locked pair 混为一谈。

### 4.3 人工审核硬门

每个候选必须逐项通过：

1. 主题保持；
2. 实体和对象保持；
3. 载体与外层形式保持；
4. 风险行为是唯一实质变化；
5. 两侧均没有回答或新增操作细节。

任何未审核候选默认 `pending_human_review`。代码允许开发 smoke 临时加载未审核样本，但会将产物标为 `formal_eligible=false`，不得进入论文结果。

## 五、CNRF 算法

### 5.1 局部箭头

对模型层 `l` 和读出 `v`，对有害/无害端点的隐藏状态分别做 L2 归一化：

\[
b_i=\phi(h(x_i^-)),\qquad q_i=\phi(h(x_i^+)).
\]

中点和风险箭头为：

\[
m_i=\frac{b_i+q_i}{2},\qquad d_i=q_i-b_i.
\]

中点用于寻找内容和载体相近的局部区域，箭头用于读取风险状态。箭头过短或中点范数接近零的对会被质量门拒绝，因为短箭头的归一化坐标会放大噪声。

### 5.2 邻域评分

测试表示 `z` 先按中点余弦距离检索最近的 `k=5` 个不同 `pack_id`。对每个近邻：

\[
r_i(z)=\frac{\|z-b_i\|_2^2-\|z-q_i\|_2^2}{\|q_i-b_i\|_2^2+\epsilon}
=\frac{2(z-m_i)^\top d_i}{\|d_i\|_2^2+\epsilon}.
\]

在无害端点分数约为 -1，在有害端点约为 +1，在两者垂直平分面上约为 0。最终分数是五个局部坐标的中位数。中位数限制少数脏配对的影响；固定裁剪只用于防止极短剩余箭头产生数值爆炸。

### 5.3 去自匹配

reference 端点用于估计每个读出的稳健中心和尺度时，必须排除自身整个 `pack_id`。验证、校准和测试都不能使用自己的箭头。否则端点天然得到约 ±1，会产生严重的自匹配泄漏。

### 5.4 文本/图文分库与多读出

- 文本输入只查询文本风险场；
- 含图输入只查询图文风险场；
- 候选读出为最后位置、非图像位置均值、图像位置均值；
- 每个读出独立选层、独立建立相同的 CNRF；
- 使用 reference 无害 leave-pack-out 分数的中位数和 MAD 做稳健标准化；
- 有效读出分数取最大值；门槛必须在融合后统一校准。

图像位置读出对纯文本无效，不得用零向量填充并参与融合。
即使图像位置读出在图文样本上形式有效，也必须通过反事实敏感性门：若同图配对的
有害/无害端点在该读出上的归一化箭头范数均低于质量阈值，则该读出应记录为
`skipped_degenerate_field`，而不是降低阈值或用零箭头参与融合。Gemma 类模型中，
图像 token 可能不受位于其后的文本变化影响，因此该检查是方法定义的一部分。

### 5.5 选层

对每个读出，在完整留出 validation `pack_id` 上：

1. 仅用 reference 包建立风险场；
2. 用 validation 的无害端点临时确定 5% FPR 分组门槛；
3. 优先选择最差 evaluation group TPR 最高的层；
4. 再依次比较总体 TPR、FPR 和 AUROC。

最终攻击不能反向改变层号、读出集合或 `k`。

### 5.6 门槛

对每个无害保护组使用有限样本 split-conformal 上尾次序统计量：

\[
\tau_g=\operatorname{OrderStat}_{\lceil(n_g+1)(1-\alpha)\rceil}(S(B_g)),
\qquad \tau=\max_g\tau_g.
\]

主工作点 `alpha=5%`，附加报告 1%。取各组门槛最大值，是满足所有经验分组 FPR 约束时最不保守的联合门槛。论文还必须报告每组样本数和 Wilson 置信区间；小校准集只能给出经验控制，不能宣传成开放世界统计保证。

### 5.7 邻域支持和弃权

最近中点距离超过 calibration 的 99% 支持半径时，预测标为 `unsupported`。CNRF 的研究输出是三态：

- `risk`：融合分数超过门槛；
- `safe`：分数低于门槛且至少一个读出有邻域支持；
- `abstain`：分数低于门槛但所有读出均缺乏支持。

`unsupported` 不是有害证据，不能在检测指标里偷偷算作成功检出。部署时若把 abstain 路由到保守安全响应，必须把无害 abstention 计入最终误拒率。

## 六、实验设计

### 6.1 两条轨道

**轨道 A：RCS 忠实复现。** 使用 RCS 发布的数据组成、投影、KCD/MCD 和门槛，回答实现是否忠实。

**轨道 B：等数据机制比较。** 所有方法使用同一反事实端点、同一 split、同一选层预算和同一无害校准集，回答差异是否来自算法。

两条轨道不能混合。轨道 A 不能用于证明 CNRF 更好，轨道 B 不能被称为 RCS 论文复现。

### 6.2 模型与基线

第一阶段冻结：

- Qwen2.5-VL-7B-Instruct；
- Gemma3-12B-IT。

等数据基线：

1. 全局平均风险箭头；
2. 原始表示普通双侧 KNN；
3. RCS-KCD；
4. RCS-MCD；
5. CNRF。

新增模型只能在方法冻结后做外部确认，不能用来继续调主方法。

### 6.3 主指标

检测层主指标：

- XSTest-safe FPR 与 95% CI；
- XSTest-unsafe TPR；
- MM-Vet FPR、abstention 和最终 utility；
- JailBreakV-28K 各载体 TPR；
- FigStep、JOOD、CS-DJ 中“基础模型原本产生有害回答”的条件检测召回；
- 总体、来源宏平均、最差来源、最差载体、support coverage。

生成层主指标：

- 各攻击单独 ASR，而非混合平均；
- 相对无防御基础模型的 ASR 降幅；
- XSTest-safe 与 MM-Vet 相对无防御新增误拒；
- abstain 路由引入的额外拒绝。

### 6.4 四个主消融

1. 普通双侧 KNN vs CNRF：检验收益是否来自配对局部坐标；
2. 全局平均箭头 vs CNRF：检验是否需要局部性；
3. 最后位置 vs 多读出融合：检验视觉攻击覆盖是否来自互补读出；
4. 平衡 F1 门槛 vs 最差无害组 conformal 门槛：检验低误报是否来自校准目标。

支持半径只作为诊断/选择性路由附加消融，不与核心几何贡献混为一个复杂模块。

### 6.5 统计协议

- 重采样、置信区间和样本效率曲线以 `pack_id` 为单位；
- 至少 10 次 reference 包子采样，报告均值、标准差和最差值；
- 主要方法差异使用相同 bootstrap 包的配对区间；
- 同时报告 pooled、source macro 和 worst source；
- 最终攻击结果出现后不得重新选层、选 `k`、改融合或改门槛；
- 历史上已查看过的 XSTest/攻击结果不能称为完全盲测，若继续迭代必须新增最终确认集。

## 七、成功、停止与否证标准

### 7.1 进入正式外部测试前

- 箭头来源/载体 macro-F1 明显低于端点与中点；
- Qwen 内部等数据比较同时改善最差组 TPR 与 FPR；
- Gemma 使用相同协议重复成立；
- 配置、层、读出、`k`、门槛协议全部冻结。

任一条件失败，先修正配对或否定假设，不增加来源对抗器、可学习融合器或复杂投影来掩盖失败。

### 7.2 论文级目标

- XSTest-safe 检测 FPR ≤ 5%；
- XSTest-unsafe TPR ≥ 95%；
- FigStep、JOOD、CS-DJ 条件攻击召回分别 ≥ 95%；
- 接入统一确定性安全响应后，各攻击 ASR ≤ 5%；
- XSTest-safe 最终误拒相对无防御增加 ≤ 2 个百分点；
- 两个模型上均成立，并能用邻域覆盖或箭头不一致解释主要失败。

这些是预注册目标，不是当前结果。

### 7.3 应主动报告的负结果

- 箭头仍高精度预测来源；
- 新载体大面积 `unsupported`；
- 不同载体的同风险箭头方向相反；
- CNRF 优势在等数据 RCS 上消失；
- 多读出只提高内部验证，不提高三种 OOD 攻击；
- 低检测 FPR 依赖大量 abstain，并在生成层造成严重误拒。

## 八、可以与不可以声称什么

在完成正式实验前，可以声称：

- 提出了一个清晰、无神经投影、可证伪的局部反事实风险场方案；
- 已实现严格 `pack_id` 隔离、去自匹配、分组门槛和来源审计；
- 算法在合成数据上满足预期几何性质。

不能声称：

- 已经优于 RCS、HiddenDetect 或 NEARSIDE；
- 已经证明来源水印被消除；
- 已经对 FigStep、JOOD 或 CS-DJ 泛化；
- 已经实现低误拒部署；
- 算法形式在全部相关文献中从未出现。

## 九、总体评价

本 proposal 最值得推进的部分，是把“来源混杂”从一个泛泛的局限变成了数据单位、几何结构和实验否证条件。CNRF 的方法本身足够简单，这反而是优点：如果它在严格等数据和低误报协议下成立，改进更容易归因于局部反事实几何；如果不成立，也能快速定位为配对不干净、邻域覆盖不足或风险方向被载体旋转。

综合判断：**建议继续，定位为 Major Revision 后可实验验证的强 proposal。** 下一阶段的首要目标不是跑齐所有攻击，而是完成少量高质量反事实包并验证 H1。只有 H1 和内部等数据 H2 同时通过，才值得投入两模型完整 benchmark。
